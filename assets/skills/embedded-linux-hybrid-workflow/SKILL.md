# Embedded Linux Hybrid Workflow

## When to Use

Use this skill when all of the following are true:
- The codebase spans multiple layers (for example: U-Boot, kernel, device tree, drivers, middleware, userspace, CI).
- You need concrete execution in terminal/CI: edit files, run build/test scripts, fix compile errors, and produce patch artifacts.
- You also want a stronger architecture explanation and design review pass from Claude based on the actual diff.

## Workflow

### Step 0: Context Snapshot (Fast Intake)

Before editing, collect a compact context snapshot:
- git branch/head/status
- key build/config files (README, CMake/Makefile, Kconfig, Yocto recipes, dts/dtsi)
- likely impacted modules

`hybrid-run.sh` does this automatically and writes `context-snapshot.md`.

### Step 1: Codex Executes Changes and CI Loop (Primary Engine)

Codex performs implementation and debugging directly in the workspace:
- apply focused code edits
- run build and test commands
- iterate until commands pass
- keep changes reviewable and minimal

Use the helper script to standardize logs:

```bash
bash scripts/run-codex-ci-loop.sh [options] "<build_cmd>" "<test_cmd>" [output_dir]
```

Example:

```bash
bash scripts/run-codex-ci-loop.sh \
  --max-retries 1 \
  --retry-delay 5 \
  "bitbake core-image-minimal -c compile -f" \
  "ctest --output-on-failure" \
  artifacts/codex-ci
```

Failure handling in this step:
- automatic retry for transient failures within retry budget
- fast-fail for deterministic failures (can disable with `--no-fast-fail`)
- classification artifacts generated by `scripts/classify-build-failure.sh`

### Step 2: Export Review Bundle from Git Diff

Create a compact review package for architecture analysis:

```bash
bash scripts/prepare-review-bundle.sh [output_dir] [base_ref]
```

Examples:

```bash
# Working tree diff
bash scripts/prepare-review-bundle.sh artifacts/hybrid-review

# Review only commits since main
bash scripts/prepare-review-bundle.sh artifacts/hybrid-review main
```

Bundle outputs include:
- `changes.diff`
- `changed-files.txt`
- `diffstat.txt`
- `module-roots.txt`
- `review-input.md`

### Step 3: Ask Claude for Architecture/Refactor Review (Secondary Engine)

Run Claude review over the generated bundle:

```bash
bash scripts/request-claude-architecture-review.sh [bundle_dir] [focus]
```

Example:

```bash
bash scripts/request-claude-architecture-review.sh \
  artifacts/hybrid-review \
  "driver-kernel-userspace coupling, ABI risk, refactor roadmap"
```

The script calls `codeagent-wrapper --backend claude` and stores markdown output in the bundle directory.

### Step 4: Codex Applies Review Feedback

Codex translates review output into actionable code work:
- prioritize high-risk findings
- implement smallest safe fixes first
- rerun build/test loop
- export updated patch bundle if needed

### One-Command Mode

For end-to-end execution in one command:

```bash
bash scripts/hybrid-run.sh \
  --build "<build_cmd>" \
  --test "<test_cmd>" \
  --base-ref main \
  --focus "ABI impact, cross-layer coupling, phased refactor roadmap"
```

This orchestrates:
1. context snapshot
2. Codex build/test loop with retry strategy
3. review bundle generation
4. Claude architecture review
5. consolidated report output

## Output Contract

At completion, provide:
- passing build/test summary (`run-codex-ci-loop` output)
- patch artifact (`changes.diff`)
- Claude architecture review markdown
- final change summary with risks and follow-up actions
- explicit assumptions and unresolved unknowns

## Guardrails

- Do not submit a patch with failing build/test unless explicitly requested.
- Keep each patch focused on one problem area when possible.
- If Claude review conflicts with repository constraints, prioritize repository reality and explain the tradeoff.
- For cross-layer embedded changes, use [references/embedded-linux-checklist.md](references/embedded-linux-checklist.md).
- For tasks spanning multiple files/layers, enumerate requirements and test scenarios (happy/edge/error/state) before claiming completion.

## Fallbacks

- If `codeagent-wrapper` or `claude` backend is unavailable, continue with Codex-only review and clearly label the missing Claude pass.
- If no git repository is detected, skip bundle export and provide plain-text file-level change summary instead.
- For transient command failures, retry once by default and log fallback decisions in summary/report files.
- `hybrid-run.sh` will classify Claude unavailability into `runtime`, `auth`, or `network` for clearer triage.
